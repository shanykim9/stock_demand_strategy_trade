# Kiwoom STG/Simul Demand

키움 REST API로 **외국인·기관 수급**을 분석해 매수 후보를 찾고, 웹 UI에서 확인한 뒤 지정 시각에 **자동 매수·매도**까지 실행하는 주식 자동매매 시스템입니다.

메인 웹 UI는 `stgdemand.py`(포트 `7791`)이며, 분석 엔진은 `simuldemand.py`를 재사용합니다.

---

## 한 줄 요약

> 테마 종목의 **수급 전환 신호**를 탐지 → **다음 거래일(또는 당일) 시가 자동 매수** → **트레일링 익절 / 손절**로 자동 매도

---

## 전체 실행 흐름

```
stgdemand.py 실행 (웹서버 :7791)
    │
    ├─ bsdemand 확장 자동 로드 (자동매수/매도 UI + 스케줄러)
    ├─ 자동 분석 스케줄러 (설정 시각에 테마 종목 일괄 분석)
    │
    ├─ [분석] 종목별 외국인·기관 순매수 + 일봉 OHLC 수집
    │         → 수급 전환 신호 탐지
    │         → 신호 종목을 매수 계획으로 저장
    │
    ├─ [매수] 09:00 이후 시가 기준 시장가 매수
    │
    └─ [매도] 보유 중 익절/손절 감시 → 트리거 다음 거래일 시가 매도
```

## 핵심 파일

| 파일 | 역할 |
|------|------|
| `stgdemand.py` | 메인 웹 UI, 자동 분석 스케줄러, MD 파일 업로드 |
| `simuldemand.py` | 수급 신호 탐지 및 익절/손절 시뮬레이션 엔진 |
| `demand.py` | 키움 API 토큰 발급, TR 호출, 종목명↔코드 해석 |
| `bsdemand.py` | 자동매수/자동매도 UI 및 실행 스케줄러 (`stgdemand` 확장) |
| `buysell.py` | 실제 주문, 호가, 계좌 처리 |
| `start_stgdemand.bat` | Windows 부팅 자동실행 (중복 실행 방지, 로그 롤링) |
| `static/styles.css` | 웹 UI 스타일 |

## 1. 웹 서버 (`stgdemand.py`)

- 브라우저 접속: `http://127.0.0.1:7791`
- **4개 테마** 관리: 반도체, 원자력_전력, 로봇, 방산
- **MD 파일 업로드**로 테마별 관심 종목 목록 갱신
- 분석 진행 상황을 **실시간 UI**로 표시
- **자동 분석 스케줄** 설정 (기본 `20:10`, 웹 UI에서 변경 가능)
- `stgdemand.py` 실행 시 `bsdemand.py` 확장이 자동 로드됨 (`STG_ENABLE_BSDEMAND=0`으로 비활성화 가능)

---

## 2. 수급 분석 (`simuldemand.py` + `demand.py`)

각 종목마다 아래 과정을 반복합니다.

| 단계 | 내용 |
|------|------|
| 데이터 수집 | 키움 API로 **외국인·기관 일별 순매수** + **일봉 OHLC** 조회 |
| 캐시 | `data/stgdemand_cache/`에 저장, 이후 **증분 업데이트** |
| 신호 판단 | 외국인·기관 모두 **2일 연속 감소 → 2일 연속 증가** 패턴이 겹치는 날 |
| 시뮬레이션 | 과거 신호 기준으로 익절/손절 수익률 계산 |

### 신호 조건

외국인과 기관이 **같은 날** 아래 패턴을 동시에 보이면 매수 후보로 판정합니다.

- **감소**: 오늘 순매수 < 전일 순매수 (2일 연속)
- **증가**: 오늘 순매수 > 전일 순매수 (2일 연속)
- **기록 날짜**: 증가 구간이 시작된 날

---

## 3. 자동매수 (`bsdemand.py`)

분석이 끝나면 신호 종목을 **매수 계획**(`data/bsdemand_buy_plan.json`)으로 저장합니다.

| 상황 | 매수 예정일 |
|------|-------------|
| 오늘 신호 발생 | **다음 거래일** 시가 매수 |
| 어제 신호 + 오늘 매수 시작 시각 전 분석 | **오늘** 시가 매수 |

### 매수 실행 규칙

- 기본 **09:00 이후** (`BSDEMAND_BUY_START_HHMM`)
- 종목당 예산(기본 50만 원, `BSDEMAND_BUDGET_PER_STOCK`)으로 **시장가 매수**
- 예산으로 1주도 살 수 없으면 주문하지 않음 (초과매수 금지)

---

## 4. 자동매도 (`bsdemand.py` + `buysell.py`)

보유 중인 종목을 주기적으로 감시합니다.

| 규칙 | 조건 | 매도 시점 |
|------|------|-----------|
| **트레일링 익절** | 최고가 대비 **-10%** (저가 기준) | 트리거 다음 거래일 **시가** |
| **손절** | 매수가 대비 **-7%** (종가 기준) | 트리거 다음 거래일 **시가** |
| **우선순위** | 둘 다 해당 시 **트레일링 우선** | |

---

## 5. 실거래 안전장치 (`buysell.py`)

기본값은 **모의(드라이런) 모드**입니다. 실제 주문이 나가려면 아래 **두 조건을 모두** 충족해야 합니다.

- `KIWOOM_DRY_RUN=0`
- `KIWOOM_ENABLE_LIVE_TRADING=YES`

---

## 하루 운영 흐름 (예시)

```
설정 시각 (예: 20:10)  →  자동 분석 실행
                       →  테마 종목 전체 수급 스캔
                       →  신호 종목 매수 계획 저장

09:00 이후             →  매수 계획 종목 시가 매수 시도

장중                   →  보유 종목 고가 추적, 익절/손절 조건 감시

익일 시가              →  조건 충족 종목 자동 매도
```

## 환경 설정

`.env.example`를 복사해 `.env`를 생성한 뒤 값을 입력합니다.

```bash
cp .env.example .env
```

### 주요 환경 변수

| 변수 | 설명 | 기본값 |
|------|------|--------|
| `APP_KEY` / `APP_SECRET` | 키움 API 인증키 | (필수) |
| `KIWOOM_ACCOUNT_NO` | 계좌번호 | (실거래 시 필수) |
| `KIWOOM_DRY_RUN` | `1`=모의, `0`=실거래 모드 | `1` |
| `KIWOOM_ENABLE_LIVE_TRADING` | `YES`일 때만 실주문 허용 | `NO` |
| `KIWOOM_DMST_STEX_TP` | 거래소 (`KRX` 등) | `KRX` |
| `KIWOOM_MARKET_OPEN` | 장 시작 시각 | `09:00` |
| `KIWOOM_LIVE_ORDER_START` | 주문 제출 시작 시각 | `09:00` |
| `KIWOOM_POLL_SEC` | 감시 주기(초) | `2.0` |
| `BSDEMAND_BUY_START_HHMM` | 자동매수 시작 시각 | `09:00` |
| `BSDEMAND_BUDGET_PER_STOCK` | 종목당 매수 예산(원) | `500000` |
| `STG_ENABLE_BSDEMAND` | 자동매수/매도 확장 로드 | `1` |

## 실행 방법

### 수동 실행

```bash
python -B stgdemand.py
```

브라우저에서 `http://127.0.0.1:7791` 접속

### Windows 부팅 자동실행

- 시작프로그램에서 `start_stgdemand_hidden.vbs` → `start_stgdemand.bat` 호출
- `start_stgdemand.bat`는 **7791 포트 중복 실행 방지** + **로그 롤링** 포함

## 데이터 / 로그 파일

실행 중 아래 파일이 생성·갱신됩니다.

| 경로 | 내용 |
